
# 公司背调与面试话术 — 执行规则

> 本文件为 AI 执行指令，不是给用户看的模板。AI 必须严格按以下 5 个 Phase 顺序执行。
> **跨平台兼容**：本文件为纯 Markdown，可直接作为以下工具的规则/指令使用：
> - **Claude Code**：使用同目录下的 SKILL.md（含 YAML 元数据）
> - **Cursor**：复制到 `.cursorrules` 或 `.cursor/rules/` 目录
> - **Trae**：复制到 `.trae/rules/` 目录
> - **Codex (OpenAI)**：复制到 Codex 的 rules/instructions 配置中
> - **其他 AI 编程工具**：直接粘贴到自定义指令/规则设置中

---

## 架构概览：5-Phase 流水线

```
Phase 0 → Phase 1.1-1.3 → Phase 1.4(受阻清单) → Phase 2 → Phase 3(🔍自检) → Phase 4(人工复核) → Phase 5
准备环境   信息搜集       输出需登录/MCP项      生成初稿   AI自查+用户确认    用户核实关键数据     终稿输出
(必须)     (必须)         (必须!!!)              (必须)    (必须!!!)          (必须)              (必须)
```

**每个 Phase 不可跳过。Phase 3 是整个规则 防止 AI 编造数据的核心机制。**

## Phase 0: 环境准备

### 0.1 读取个人信息

**第一步**：使用文件读取功能读取 `profile.md`（与本文件同目录）。
- 如果文件存在 → 解析 `| **字段名** | 值 |` 格式的表格，提取所有字段值
- 如果文件不存在 → 执行 0.2

### 0.2 创建 profile 引导（仅当 profile.md 不存在时）

向用户输出以下内容，**并等待用户填写完成后再继续**：

```
⚠️ 未找到 profile.md（个人信息文件）。

为了让面试话术和 JD 匹配点更加精准，请先填写你的基本信息。
我已经从 profile.template.md 为你准备好了模板，请编辑 profile.md 填入真实信息：

| 字段 | 你需要填写的内容 |
|------|----------------|
| 姓名 | （你的真实姓名） |
| 学历 | （如：本科 / 硕士） |
| 学校 | （你的学校名称） |
| 证书 | （如：英语六级 / 其他证书） |
| 工具技能 | （如：剪映、PS、AI工具） |
| 实习经历 | （如：XX公司 XX岗位XX实习） |
| 个人项目 | （如：运营小红书账号） |
| 核心能力 | （3-4项，如：内容创作、数据复盘） |
| 职业目标-薪资 | （期望无责底薪） |
| 职业目标-休息 | （双休/大小周） |
| 职业目标-方向 | （如：深耕内容→品牌方向） |

填写完成后回复"好了"，我将继续。
```

**如果用户明确拒绝填写**：使用占位值（"申请人"、"XX大学"、"XX经验"）继续生成，并在报告开头注明"⚠️ 未提供个人信息，面试话术为通用模板，建议填写 profile.md 后重新生成"。

### 0.3 收集公司 & 岗位信息

从用户输入中提取：

**必需信息（缺少任一项必须追问）**：
1. 公司全称（或 Boss直聘显示名）
2. 岗位名称
3. 薪资范围
4. 工作地点（城市）

**可选信息（有则更精准，无则跳过）**：
5. JD 详情（岗位职责 + 任职要求）
6. 福利标签（双休/大小周/五险一金/年终奖等）

**追问模板**（当缺少必需信息时）：
```
还需要以下信息才能开始背调：
- [ ] 公司全称？（例如：上海新炬网络信息技术股份有限公司）
- [ ] 投递的岗位名称？（例如：文案策划，双休）
- [ ] 薪资范围？（例如：5-7K·13薪）
- [ ] 工作城市？（例如：广州）
```

### 0.4 确认输出范围

向用户确认：
```
我将生成完整的10模块背调报告（约15000-30000字），包括：

一 公司基本信息     二 求职者关注点      三 业务健康度
四 风险预警         五 行业深度分析      六 账号矩阵分析（含平台AI工具建议）
七 信息可信度总结   八 整体评估与建议    九 面试话术整理
十 深度思考

如果你只需要部分模块，请告诉我（例如："只要一四八九"）。
确认生成全部模块请回复"OK"。
```

## Phase 1: 信息搜集

### 1.1 搜索策略（按优先级执行）

**不要一次性搜所有平台**。按以下顺序分步搜索，每搜完一个来源，立即整理数据，再搜下一个：

| 步骤 | 来源 | 搜索方式 | 搜索词模板 | 目标数据 |
|------|------|----------|-----------|---------|
| 1 | 百度 | 联网搜索 | `"{公司全称}" 工商信息 天眼查` | 信用代码、法定代表人、注册资本 |
| 2 | 百度 | 联网搜索 | `"{公司全称}" 企查查 股东` | 股权结构、实际控制人 |
| 3 | 百度 | 联网搜索 | `"{公司简称}" site:36kr.com` | 融资报道、创始人背景 |
| 4 | Boss直聘 | 联网搜索 | `"{公司简称}" 招聘 Boss直聘 {城市}` | 在招岗位、薪资范围 |
| 5 | 脉脉 | 联网搜索 | `"{公司简称}" 工作体验 加班` | 员工匿名评价 |
| 6 | 小红书 | 联网搜索 | `"{公司简称}" 面试 公司怎么样` | 面试经验、公司口碑 |
| 7 | 知乎 | 联网搜索 | `"{公司简称}" 工作 评价` | 公司口碑评价 |
| 8 | 百度 | 联网搜索 | `"{公司简称}" 被执行 司法案件` | 司法风险 |
| 9 | 百度 | 联网搜索 | `"{行业关键词}" 行业报告 2025 2026` | 行业趋势 |

**搜索名称变体规则**：
- 同时搜全称 + 简称（去地域前缀和公司后缀）
- 示例："上海新炬网络信息技术股份有限公司" → 同时搜 "新炬网络信息技术"、"新炬"
- 如果用户提供了 Boss显示名，也作为搜索词

### 1.2 数据提取与交叉验证

**从搜索结果中提取结构化数据时，遵循以下规则**：

**规则 1 — 宁缺毋滥**：
- 搜不到的数据 → 标注【待核实】，**绝不编造具体数字**
- 示例：搜不到注册资本 → 填 `【待核实】`，不要填 `约1000万元`

**规则 2 — 矛盾检测**：
- 多个来源给出不同数据时 → 标注【存疑】+ 列出所有来源的说法
- 示例：`参保人数 36人【企查查】vs Boss标注"100-499人"【Boss直聘·存疑】`

**规则 3 — 来源优先级**：
- 工商数据：天眼查 = 企查查 > 爱企查 > 百度摘要 > AI推测
- 薪资数据：Boss直聘岗位页 > 猎聘 > 其他招聘平台 > 第三方推测
- 员工评价：脉脉 > 小红书 > 知乎 > 第三方推测
- 媒体报道：36氪 > 其他科技媒体 > 官网宣称

**规则 4 — 来源URL记录（强制执行）**：
- **每次搜索获取到关键数据时，必须同时记录该数据所在的网页URL**
- 关键数据定义：工商信息（注册资本/法定代表人/信用代码/成立时间）、融资信息、司法案件、官方社媒账号
- URL 格式：如果搜索工具返回了完整URL，直接记录；如果返回的是搜索摘要，记录搜索关键词和来源网站
- 无法获取URL的数据，在可信度标注中说明"仅搜索摘要，无直接链接"
- 示例标注：`1000万元【天眼查·https://www.tianyancha.com/company/xxxxx】`
- **目的**：确保用户可以通过点击链接直接核实信息源，而不是只能看到模糊的平台名称

### 1.3 搜索失败处理

**如果某个来源完全搜不到结果**：
- 在报告中标注 `【未搜到】该来源无相关信息`
- 不要跳过该模块 — 保留模块框架但标注数据缺失
- 在 七、 信息可信度总结 中降低对应维度的可信度星级

**如果所有来源都搜不到有效信息**：
- 向用户汇报：`"该公司公开信息极少，可能为初创企业/空壳公司/信息未上网。这是我能生成的报告（仅基于有限信息），建议面试时重点关注以下问题：[list]"`
- 仍按 10 模块输出，但大量标注【待核实】和【存疑】

### 1.4 搜索受阻清单（搜索完毕后必须输出）

> ⚠️ 本步骤的目的是：在生成报告之前，明确告知用户哪些信息受限于工具/权限无法获取，让用户决定是否需要手动补充。

**搜索全部完成后（Phase 1 最后一步），AI 必须输出以下清单**：

```markdown
## 🔒 搜索受阻清单

以下信息因工具限制或网站权限无法直接获取，需要你手动操作：

### 需要登录网站查询的
| 信息项 | 受阻原因 | 建议操作 |
|--------|---------|---------|
| 工商详细信息（注册资本/法人等） | 天眼查/企查查需要登录 | 打开 https://www.tianyancha.com 搜索"{公司全称}" |
| 投融资详情 | 企查查投融资板块需要登录 | 打开 https://www.qichacha.com 查看 |
| 司法案件详情 | 天眼查司法风险需要登录 | 同上 |

### 需要MCP/浏览器工具才能获取的
| 信息项 | 受阻原因 | 建议操作 |
|--------|---------|---------|
| Boss直聘实时薪资 | 招聘页面需JS渲染 | 手动打开Boss直聘搜索"{公司简称}"|

### 需要你补充提供的信息
| 信息项 | 说明 | 补充方式 |
|--------|------|---------|
| 完整JD | 未在搜索中找到完整JD | 复制Boss直聘岗位页JD文本 |
| 福利详情 | 公开页面未展示完整福利 | 提供面试沟通中HR告知的福利 |

> 以上信息可先跳过，我会在报告中标注【待核实】。你可以在收到报告后再补充修改。
```

**触发条件**（满足任一即输出对应行）：
- 天眼查/企查查搜索结果需要登录才能查看详情 → 输出到"需要登录网站查询的"
- 招聘页面需要JS渲染且无对应MCP工具 → 输出到"需要MCP/浏览器工具"
- 用户未提供完整JD或福利信息 → 输出到"需要你补充提供的信息"
- 微博/公众号等平台内容需要登录查看 → 输出到"需要登录网站查询的"

**输出时机**：在 Phase 1 搜索全部完成后、Phase 2 生成初稿之前。如果不需要用户操作（所有信息均可直接获取），则简化为一行提示："✅ 本次搜索未遇到登录墙限制，所有信息均可直接获取。"

## Phase 2: 生成初稿

### 2.1 文件命名与存放

- 文件名：`{工商注册全称}_{YYYY-MM-DD}_公司背调.md`
- 存放在：用户当前工作目录（或用户指定的目录）
- **加日期前缀的原因**：防止同一公司多次背调时互相覆盖，同时方便追踪背调时间线

### 2.2 文件头部

```markdown
# {公司全称} — 公司背调与面试话术

> 背调对象：{公司全称}（旗下品牌：{品牌名}）
> Boss显示名：{Boss上的公司名}
> 统一社会信用代码：{代码，搜不到填【待核实】}
> 背调日期：{YYYY-MM-DD，取当前日期}
> 信息来源：用户提供的岗位信息【招聘平台】+ 公开信息搜索
> 岗位：{岗位名称}（{福利标签}）{薪资范围}
```

### 2.3 各模块生成细则

#### 一、 公司基本信息

**工商信息表**（19个字段，不可缩减）：

| 字段 | 生成规则 |
|------|---------|
| 公司全称 | 以天眼查/企查查为准 |
| 旗下品牌 | 从商标信息提取，标注【企查查·商标信息】 |
| 总部地址 | 工商注册地址 |
| 实际办公地 | 从招聘平台获取的实际办公地址 |
| 成立时间 | 精确到日，搜不到填【待核实】 |
| 公司规模 | `参保人数【天眼查】+ Boss标注人数【Boss直聘】`，不一致时标注【存疑】 |
| 注册资本 | **必须精确**，搜不到填【待核实】 |
| 实缴资本 | 搜不到填【待核实】 |
| 企业类型 | 如"其他有限责任公司" |
| 法定代表人 | 搜不到填【待核实】 |
| 监事 | 有空则填 |
| 英文名 | 有空则填 |
| 曾用名 | 有变更记录则填 |
| 官网 | 搜不到留空 |
| 所属行业 | 国标行业分类 + 实际经营业务 |
| 登记机关 | 如"广州市番禺区市场监督管理局" |
| 经营状态 | 存续/注销/吊销 |
| 纳税人资质 | 从启信宝获取 |
| 企业规模 | 从启信宝获取（与公司规模不同维度） |

**必须生成的子模块**（即使信息不完整也要输出框架）：
1. **核心业务与服务范围** — 用3-8个要点描述，每条标注来源
2. **实际控制人/股东背景** — 股权结构表（序号/股东名称/持股比例/备注/可信度） + 实际控制人推断
3. **创始人/核心团队** — 按人分条，包含背景履历、关联企业
4. **最新融资情况/上市状态** — 融资时间线、投资方列表
5. **近2年重大业务调整** — 时间+事件+可信度 表格，至少列3条（搜不到填"近2年无公开重大业务调整信息【待核实】"）
6. **知识产权** — 商标/专利/软著 数量表

#### 二、 求职者关注点

**JD能力拆解表**（核心！每个JD维度必须匹配置化点）：

```
| JD 维度 | 核心能力要求 | 简历匹配点 |
|---------|-------------|-----------|
| （从JD原文提取） | （从JD原文提取） | （基于profile.md的真实经历） |
```

**匹配规则**：
- 有 profile.md → 每个 JD 维度都要从 profile 中找到对应经历/技能 → 写具体匹配点
- 无 profile.md → 每个 JD 维度都写"待补充个人信息后匹配【需填写profile.md】"
- 如果 JD 中某个要求 profile 中确实没有 → 标注 `⚠️ 经验缺口：该维度暂无直接匹配经历，建议面试时用[XX方式]弥补`

**薪资对比表**（必须横向对比同公司多个岗位）：

| 岗位 | 平台 | 经验要求 | 学历 | 薪资 | 双休 | 状态 |
|------|------|----------|------|------|------|------|

- 至少列出 3 个同公司不同岗位
- 如果只搜到 1 个岗位 → 标注 `【待核实】该公司其他岗位薪资信息未搜到，建议在Boss直聘上搜索"{公司简称}"查看更多`

**加班文化/工作时间**（必须包含矛盾信息对比）：
- 分"已确认"和"【存疑】"两部分
- 给出面试时确认加班情况的 3-5 个具体问题

**员工福利**：
- 分"已确认福利"（标注来源）和"存疑·需面试确认的福利"两部分

#### 三、 业务健康度

- 主力产品/服务说清楚"卖什么、卖给谁、怎么赚钱"
- 现金流推测：标注【第三方推测】+ 推断依据
- 最后给出优势 list 和风险 list，各至少 3 条

#### 四、 风险预警

**必须查**：
- 司法案件数量（从搜索结果提取，搜不到填"未搜到相关司法案件信息【待核实】"）
- 行政处罚记录
- 经营异常记录
- 被执行人记录

**每个风险项给出评级**：🔴高风险 / 🟡中风险 / 🟢低风险

#### 五、 行业深度分析

- 行业上下游用 ASCII 树形图表示
- 未来趋势至少列 3 条，每条标注【行业报告】或【第三方推测】
- 目标公司在行业中的地位：行业定位 + 差异化优势(至少3条) + 明显劣势(至少2条)

#### 六、 账号矩阵分析

**判断是否需要展开**：
- 岗位名称包含"内容/新媒体/社媒/运营/品牌/营销/推广/文案/视频" → 深入展开
- 其他岗位 → 仅列一个简表（平台+账号名+链接），不深入分析

**必须包含的子模块**（展开模式）：
1. 品牌定位表
2. 账号矩阵布局表（平台/账号名称/账号链接/定位/内容策略/现状评估）
3. 内容策略深度分析
4. 问题诊断与风险预警表
5. 对求职者的意义
6. 账号数据汇总表

**新增：平台AI分析工具建议**（展开模式下必须包含）：
> 在对各平台进行内容策略分析时，建议结合该平台对应的AI分析工具获取更深度的数据洞察，弥补人工搜索无法覆盖的分析维度。

| 平台 | 对应AI分析工具 | 可获取的增量数据 | 使用建议 |
|------|--------------|----------------|---------|
| 抖音 | **豆包（字节系AI）** | 视频播放趋势、热门话题标签、用户画像分布、竞品账号对比 | 输入"分析{账号名}的抖音内容策略" |
| 微信/公众号 | **元宝（腾讯系AI）** | 公众号文章传播路径、读者画像、阅读完成率、最佳发布时间 | 输入"分析{公众号名}的读者画像和传播效果" |
| 小红书 | **点点（小红书官方AI）** | 笔记SEO关键词、爆文规律、评论区用户需求挖掘、品牌词云 | 输入"分析{品牌名}在小红书的内容表现" |
| B站 | **B站AI分析工具** | 视频完播率、弹幕情感分析、粉丝增长拐点 | 搜索"{账号名} B站数据分析" |
| 快手 | **快手AI创作助手** | 同城流量分布、老铁互动数据、直播转化漏斗 | 输入"快手{账号名}运营分析" |
| 微博 | **微博数据分析工具** | 热搜关联度、话题传播链路、大V互动关系 | 搜索"{品牌名} 微博数据分析" |

**输出格式**（在模块六末尾追加）：

```markdown
### 💡 平台AI工具补充分析建议

除了以上基于公开信息的人工分析，建议你使用各平台对应的AI分析工具获取更深度的数据：

| 平台 | 推荐AI工具 | 建议分析内容 | 为什么需要AI工具 |
|------|-----------|-------------|-----------------|
| {如果公司有抖音账号} | 豆包 | {具体分析点} | 抖音视频数据仅后台可见 |
| {如果公司有公众号} | 元宝 | {具体分析点} | 公众号读者画像需AI挖掘 |
| {如果公司有小红书账号} | 点点 | {具体分析点} | 爆文规律需AI语义分析 |

> 以上AI工具可帮助你在面试时展示出超越公开信息的深度洞察，建议提前准备1-2个AI分析发现作为面试加分项。
```

#### 七、 信息可信度总结

**必须严格按此星级定义**：

| 可信度 | 定义 | 示例 |
|--------|------|------|
| ★★★★★ | 三源以上交叉验证的工商/上市公开数据 | 天眼查+企查查+启信宝一致的法定代表人 |
| ★★★★ | 双源验证或官方公开披露 | 上市公司年报、36氪+企查查一致的融资信息 |
| ★★★ | 单源数据或可合理推断 | Boss直聘薪资、行业报告趋势 |
| ★★ | 匿名/主观来源 | 脉脉员工评价 |
| ★ | 纯推测或无法验证 | AI推测的现金流 |

#### 八、 整体评估与建议

**必须包含**：
- 公司类型（一句话定位，15字以内）
- 岗位类型（一句话定位，15字以内）
- 适合人群（✅ 至少 3 条）
- 不适合人群（❌ 至少 2 条）
- 关键风险点（⚠️ 至少 3 条，面试时主动确认）
- 优势亮点（💡 至少 3 条）
- 总结判断（一段话，80-150字，包含：综合评价 + 是否建议投递 + 面试重点确认事项）

**判断"是否建议投递"的依据**：
- 公司经营状态为"存续" + 无重大司法风险 → 通常建议投递
- 公司有大量被执行/经营异常 → 标注"谨慎投递，建议面试时确认"
- 公司已注销/吊销 → 标注"不建议投递"

#### 九、 面试话术整理

**自我介绍**必须包含 6 个组件：
1. 开场钩子（30秒版）— 用一句话抓住面试官注意力，结合对公司的了解
2. 融合优化版（1分钟）— 背景+能力+为什么选这家公司，包含具体数字
3. 精简速记卡（ASCII方框图）— 面试前5分钟快速记忆
4. 记忆口诀 — 4句顺口溜，方便背诵
5. 关键词对照表 — 你的关键词↔公司关键词 对照
6. 应急卡壳救场话术 — 忘词时的3个救场句子

**"你为什么选择我们公司"** — 必须体现：
- 对公司的深度了解（引用背调中的具体信息）
- 个人经历与公司业务的结合点
- 不少于 100 字

**"你了解我们公司吗"** — 必须体现：
- 5-8 个具体信息点（从背调报告中提取）
- 有数据，有洞察，不只是背书
- 最后加一个"我特别注意到...这也是我想加入的原因之一"

**个性化 Q&A** — 5 类各至少 1 题，每题含完整话术（不少于 80 字）：
- 岗位理解类
- 运营思路类
- 行业看法类
- 经验不足类
- 加班/压力类

**反向提问** — 4 类各至少 3 个问题（共 12+ 个）：
- 关于公司/团队
- 关于岗位/业务
- 关于薪酬/发展
- 关于工作环境/氛围

#### 十、 深度思考与待确认问题

**必须体现独特洞察，不能是泛泛而谈的模板话术**。至少 4 条思考，每条都要结合该公司/该岗位的具体情况。

## Phase 3: 🔍 AI 自检 — 防编造数据机制（核心！不可跳过！）

> **这是本规则 区别于普通模板的关键环节。AI 必须在输出给用户之前，先检查自己的输出是否存在编造/幻觉。**

### 3.1 逐条自检

**检查清单**：

| 检查项 | 自检问题 | 可疑特征 | 处置 |
|--------|---------|---------|------|
| 注册资本 | "这个数字是我搜到的还是我编的？" | 整数如"1000万元"、约数如"约5000万" | 改为【待核实】 |
| 法定代表人 | "这个人名是真实的吗？" | 无法从搜索结果中确认 | 改为【待核实】 |
| 成立时间 | "这个日期精确吗？" | 只搜到年份，却写了月日 | 只保留年份 |
| 参保人数 | "这个数字精确吗？" | 整数、与Boss标注矛盾 | 标注来源+【存疑】 |
| 融资轮次 | "融资信息是真实的吗？" | 没有36氪/企查查来源支撑 | 标注【第三方推测】 |
| 员工评价 | "这段评价是真实的吗？" | 没有脉脉/小红书搜索结果支撑 | **删除！不编造评价！** |
| 司法案件数 | "这个数字是真实的吗？" | 没有天眼查结果支撑 | 标注【待核实】 |
| 薪资数据 | "这个薪资是真实的吗？" | 不是从Boss直聘/猎聘页面提取的 | 标注【招聘平台·待核实】 |
| 来源URL | "关键数据有可点击的核实链接吗？" | 工商/融资/司法数据无具体URL | 补充【搜索摘要·无链接】说明 |

### 3.2 生成自查报告

**在报告末尾追加以下自查章节**（用户核实后删除）：

```markdown
## 🔍 AI 自检报告

> ⚠️ 以下为 AI 对自己的输出进行的幻觉检测结果。请优先核实 🔴 高风险项。

| 模块 | 幻觉风险 | 可疑内容 | 为什么可疑 | 核实方式 |
|------|:--------:|----------|-----------|---------|
| 一、 注册资本 | 🔴 高 | `{具体数字}` | 搜索结果中未找到精确数字 | 登录企查查核实 |
| 一、 法定代表人 | 🔴 高 | `{具体人名}` | 仅百度摘要提及，未在企查查确认 | 登录天眼查核实 |
| 四、 司法案件 | 🟡 中 | `X条案件` | 基于搜索片段推测数量 | 登录天眼查核实 |
| 九、 员工评价 | 🟡 中 | 未编造 | — | ✅ 已自查通过 |

**自检总结**：
- 总断言数：{N}条
- 已确认真实：{X}条
- 标注【待核实】：{Y}条
- 标注【第三方推测】：{Z}条
- ⚠️ **建议用户在面试前优先核实以上 🔴 高风险项**
```

### 3.3 高危数据自动降级

**以下类型的数据，AI 生成时必须自动降级处理**：

| 数据类型 | 降级规则 |
|---------|---------|
| 精确数字（注册资本、营收、持股比例） | 无明确搜索结果 → 填【待核实】，不写具体数字 |
| 人名（法定代表人、CEO） | 无2个以上来源确认 → 填【待核实】 |
| 日期（成立时间、融资时间） | 无明确搜索结果 → 填【待核实】 |
| 员工评价内容 | **绝不编造具体评价**，只写"脉脉/小红书未搜到相关评价" |
| 司法案件数量 | 无明确搜索结果 → 填"未搜到相关信息【待核实】" |

## Phase 4: 人工复核

### 4.1 输出复核指引

生成报告后，**必须在报告末尾输出以下复核指引**：

```markdown
## ⚠️ 信息验证清单

> 以下信息由 AI 生成，**部分内容可能存在幻觉**，请核实。

### 🔴 必须核实（工商核心数据）

| 信息 | 报告中的值 | 核实方式 | 你的核实结果 |
|------|-----------|----------|-------------|
| 统一社会信用代码 | `{值}` | 打开天眼查搜索"{公司全称}" | |
| 法定代表人 | `{值}` | 同上，查看工商信息页 | |
| 注册资本 | `{值}` | 同上 | |
| 实缴资本 | `{值}` | 打开企查查搜索"{公司全称}" | |
| 成立时间 | `{值}` | 天眼查/企查查 | |
| 经营状态 | `{值}` | 天眼查/企查查 | |

### 🟡 建议核实（薪资/评价）

| 信息 | 报告中的值 | 核实方式 | 你的核实结果 |
|------|-----------|----------|-------------|
| 岗位薪资 | `{值}` | 打开Boss直聘搜索"{公司简称}" | |
| 员工评价 | 报告中引用的评价 | 打开脉脉/小红书搜索"{公司简称} 工作体验" | |

```

### 4.2 等待用户反馈

- 输出报告后 **不要自动结束**，等待用户核实结果
- 用户提供修正数据后 → 进入 Phase 5
- 用户确认"无需核实" → 保留验证清单在报告中，供面试前参考

## Phase 5: 终稿输出

### 5.1 更新报告

收到用户核实的修正数据后：
1. 替换对应字段的值为核实后的真实数据
2. 将可信度标注从【待核实】/【第三方推测】改为【天眼查/企查查】等
3. 重新生成受影响模块的内容（如薪资变化会影响八、评估）

### 5.2 清理验证清单

- 所有 🔴 项已核实 → 删除验证清单章节
- 仍有未核实项 → 保留验证清单，标注"已核实 X/Y 项"
- 删除 AI自检报告 章节

### 5.3 最终确认

输出：
```
✅ 背调报告已完成并核实。

文件：{文件名}
模块：一二三四五六七八九十（共10个）
数据完整性：已核实 {X}/{Y} 项关键数据

面试前建议：
1. 熟读九、面试话术，特别是自我介绍（1分钟版）
2. 把八、中的"关键风险点"记下来，面试时主动确认
3. 把十、中的"待确认问题"准备好，在反问环节提出
```

## 信息可信度标注规范（速查表）

| 标注 | 含义 | 使用条件 |
|------|------|---------|
| 【天眼查/企查查】 | 工商信息，双源确认 | 在天眼查和企查查均能查到 |
| 【启信宝】 | 启信宝企业数据 | 在启信宝查到 |
| 【Boss直聘·招聘平台】 | Boss直聘在招岗位数据 | 在Boss直聘搜到 |
| 【拉勾网】 | 拉勾网招聘/评价数据 | 在拉勾网搜到 |
| 【36氪】 | 36氪媒体报道 | 在36氪搜到相关报道 |
| 【官网宣称】 | 公司官网自述 | 官网有明确描述 |
| 【官网宣称·存主观色彩】 | 官网的营销话术 | 官网有夸大成分的描述 |
| 【招聘平台】 | Boss直聘/猎聘等综合 | 多平台交叉信息 |
| 【员工评价】 | 脉脉/小红书/知乎匿名评价 | 搜到实际员工发言 |
| 【行业报告】 | 艾瑞/36氪等第三方报告 | 引用行业研究报告 |
| 【媒体报道】 | 综合媒体报道 | 多个媒体来源 |
| 【第三方推测】 | AI基于公开信息的合理推断 | 没有直接来源，但可通过逻辑推断 |
| 【存疑】 | 信息矛盾或来源可疑 | 多源数据不一致 |
| 【待核实】 | 需要人工核实的数据 | 搜索未找到精确信息 |
| 【未搜到】 | 该来源无相关信息 | 搜索了但没有结果 |
| 【AI生成·待核实】 | AI基于训练数据生成 | 非实时搜索，可能过时或错误 |
| 【来源链接】 | 附带可直接访问的核实URL | 搜索工具返回了完整网页链接，用户可点击直达 |
| 【搜索摘要·无链接】 | 基于搜索片段提取，无直接URL | 信息来自搜索引擎摘要，未获取完整页面链接 |

## 隐私保护规则（强制执行）

1. **profile.md 中读取的所有个人信息仅用于生成面试话术**，不得写入任何可分享的文件中
2. **RULES.md 和 README.md 中不得出现任何真实个人信息**
3. **生成的背调报告中的面试话术可包含姓氏**，但不包含全名、具体学校名等可定位信息
4. **用户分享规则文件时，提醒用户确认 profile.md 已在 .gitignore 中**
5. ⚠️ **生成的背调报告（.md 文件）包含目标公司的真实商业分析数据**，请勿上传至公开仓库或分享给第三方。建议将 `*_公司背调.md` 加入你工作目录的 .gitignore

---

## 禁止行为（红线）

**AI 在执行本规则 时，以下行为绝对禁止**：

1. ❌ **编造具体数字**（注册资本、营收、持股比例、司法案件数量）
2. ❌ **编造员工评价**（脉脉/小红书的具体帖子内容）
3. ❌ **编造人名和头衔**（法定代表人、CEO、创始人姓名）
4. ❌ **编造融资时间线**（没有36氪/企查查来源支撑的融资事件）
5. ❌ **跳过 Phase 3 自检** — 这是防编造的核心机制
6. ❌ **在未核实的情况下给出确定性结论**（如"这家公司很靠谱，放心去"）
7. ❌ **将 profile.md 中的信息写入可分享文件**
